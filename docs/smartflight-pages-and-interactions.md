# SmartFlight 页面与操作逻辑设计

> 已弃用：本文档是旧版界面设计文档，仅保留为历史参考。当前 UI、布局、配色和用户操作逻辑以 `docs/smartflight-ui-redesign-no-tabs.md` 为准。

## 1. 设计原则

- 所有核心功能都围绕“高级权限已可用”前提设计。
- 页面层级尽量浅，核心动作不超过两次点击到达。
- 状态必须可解释，用户能明确知道当前为什么联网、为什么断网、为什么未执行。
- 联网列表、黑名单、白名单的编辑应尽量列表化，不使用隐蔽的层层弹窗。
- 高风险动作要可撤销或至少可追踪。

## 2. 全局结构

建议使用 5 个主页面，底部导航切换：

1. 首页
2. 应用
3. 规则
4. 主题
5. 诊断

此外增加 3 个非底部导航流程页面：

1. 首启引导 / 高级权限接入页
2. 应用选择页
3. 日志详情页

## 3. 首启引导 / 高级权限接入页

### 3.1 页面目标

- 检查 Shizuku / ADB / Root 是否至少有一种可用
- 其中 ADB 仅指“已完成初始化所需能力链”，不表示 App 长期依赖电脑在线
- 使用情况访问权限也应完成授权，否则不进入完整自动化流程
- 在不可用时阻止用户进入核心功能
- 引导用户完成最低接入条件

### 3.2 页面内容

- 品牌标题
- 当前环境状态卡片
- 三种高级接入方式说明：
  - Shizuku
  - ADB
  - Root
- 当前检测结果
- 刷新状态按钮
- 进入应用按钮

### 3.3 操作逻辑

进入 App 时：

1. 检查是否已完成接入
2. 若未完成，进入此页
3. 若已完成，直接进入首页

点击接入项时：

1. 展开说明
2. 显示检测条件
3. 显示当前结果
4. 显示“重新检测”

点击“进入应用”时：

- 仅当检测到至少一种高级执行方式可用，且使用情况访问权限已授权时允许进入

### 3.4 状态文案建议

- Shizuku 未运行
- Shizuku 已连接
- Root 已授权
- 未检测到可用高级权限
- 已满足运行条件
- 使用情况访问未授权
- 已满足完整自动化运行条件

## 4. 首页

### 4.1 页面目标

- 展示当前自动化状态
- 提供一眼可见的总开关
- 呈现当前规则命中结果和最近动作
- 提供进入关键设置页的快捷入口

### 4.2 页面模块

#### A. 顶部概览卡

显示：

- 总开关状态
- 当前控制模式：飞行模式 / 移动数据
- 当前执行器：Shizuku / ADB / Root
- 当前统一网络状态

操作：

- 点击总开关可启停自动化
- 点击当前模式可跳转规则页

#### B. 当前状态卡

显示：

- 当前前台应用
- 屏幕状态
- 当前是否连接 Wi-Fi
- 当前 Wi-Fi 状态
- 当前蓝牙状态

#### C. 规则命中卡

显示：

- 当前是否命中联网列表
- 当前是否命中黑名单
- 当前命中来源是自动还是手动
- 当前是否因 Wi-Fi 例外规则跳过动作
- 下一次待执行动作

#### D. 最近动作卡

显示：

- 最近一次执行时间
- 执行动作
- 触发原因
- 执行结果

操作：

- 点击进入日志详情页

#### E. 快捷入口区

- 管理应用
- 编辑规则
- 打开诊断

### 4.3 操作逻辑

总开关关闭时：

- 停止自动化规则调度
- 停止前台服务
- 首页上仍可查看最近一次运行快照
- 所有自动动作暂停
- 当前前台应用、规则命中和待执行动作均标记为“非实时”

总开关开启时：

1. 检查高级执行器是否可用
2. 若不可用，给出错误状态并引导到诊断页
3. 若可用，恢复规则监控

若存在多个可用执行器：

1. 先读取用户指定的首选执行器
2. 若首选不可用，则按默认优先级自动回退
3. 将最终实际使用的执行器显示在首页

## 5. 应用页

### 5.1 页面目标

- 管理联网列表
- 管理白名单
- 管理黑名单
- 以最低学习成本完成联网/非联网切换

### 5.2 页面结构

页面顶部：

- 搜索框
- 重新扫描按钮
- 筛选切换

筛选标签建议：

- 全部
- 联网
- 非联网
- 白名单
- 黑名单

列表项内容：

- 应用图标
- 应用名
- 包名
- 联网状态
- 来源标签：自动 / 手动
- 快捷操作按钮

### 5.3 列表项操作

每个应用项提供：

- 设为联网
- 设为非联网
- 恢复默认

推荐交互方式：

- 默认显示主要动作按钮
- 次级动作放在右侧更多菜单

### 5.4 页面逻辑

首次进入时：

1. 读取本地应用数据库
2. 若首次扫描未完成，则触发扫描
3. 扫描完成后展示结果

点击“重新扫描”时：

1. 重新读取已安装应用
2. 根据规则更新自动识别结果
3. 同步联网列表中的自动项
4. 保留用户对白名单和黑名单的手动修改
5. 对已卸载应用做清理

### 5.5 联网状态与来源标签

列表对用户只展示两种主状态：

- 联网
- 非联网

列表项可以显示一个来源标签：

- 自动
- 手动

实际展示效果如下：

- `联网 + 自动`：应用被扫描识别为需要联网，当前在联网列表中
- `联网 + 手动`：应用被用户手动设为联网，当前在白名单中，同时也在联网列表中
- `非联网 + 手动`：应用被用户手动设为非联网，当前在黑名单中，且不在联网列表中
- `非联网`：默认状态，没有被自动识别为需要联网，也没有被用户手动标记

### 5.6 联网操作逻辑

将应用设为联网后：

- 加入白名单
- 加入联网列表
- 若同时在黑名单中，则自动移出黑名单
- 列表显示为 `联网 + 手动`

### 5.7 非联网操作逻辑

将应用设为非联网后：

- 加入黑名单
- 从联网列表移出
- 若同时在白名单中，则自动移出白名单
- 列表显示为 `非联网 + 手动`

### 5.8 恢复默认逻辑

恢复默认后：

- 移出白名单
- 移出黑名单
- 若当前扫描命中，则加入联网列表并显示为 `联网 + 自动`
- 若当前扫描未命中，则从联网列表移出并显示为 `非联网`

## 6. 应用选择页

### 6.1 页面目标

- 用于从规则页或首页快捷指定应用联网或非联网

### 6.2 页面内容

- 搜索框
- 应用列表
- 当前目标模式标题：
  - 选择设为联网的应用
  - 选择设为非联网的应用

### 6.3 操作逻辑

从外部进入时传入模式：

- `SelectOnline`
- `SelectOffline`

点击应用项后：

1. 执行加入操作
2. 立即返回上一级
3. 显示简短提示

## 7. 规则页

### 7.1 页面目标

- 集中配置自动化行为
- 让规则间关系可理解

### 7.2 页面分区

#### A. 联网控制模式

选项：

- 使用飞行模式
- 使用移动数据

说明：

- 两者都要求高级权限

#### B. 息屏规则

选项：

- 息屏后自动断网
- 延迟秒数输入
- 在 Wi-Fi 环境下不自动执行断网动作

逻辑：

- 若关闭“息屏后自动断网”，则该分区其他控件部分禁用

#### C. 前台应用规则

选项：

- 启动目标应用时自动恢复联网
- 离开目标应用后延迟断网
- 延迟秒数输入
- 在 Wi-Fi 环境下不自动执行恢复联网动作
- 亮屏后不自动恢复联网
- 解锁后不自动恢复联网

#### D. 应用来源规则

选项：

- 使用联网列表

逻辑：

- 联网列表中的应用可以触发恢复联网
- 白名单和自动识别结果共同决定联网列表的组成

#### E. 状态保留规则

选项：

- 切换时保留 Wi-Fi 状态
- 切换时保留蓝牙状态

说明：

- 即使使用移动数据模式，这两项仍然保留在配置中
- 若移动数据模式本身未改动 Wi-Fi 或蓝牙，则此两项在该次动作中视为无额外执行

### 7.3 规则页操作逻辑

每次修改后：

1. 立即写入设置存储
2. 通知运行时规则引擎刷新
3. 若改动影响当前状态，可立即重新评估

例如：

- 打开“在 Wi-Fi 环境下不自动执行断网动作”后，若当前处于已连接 Wi-Fi 且待断网倒计时中，应取消该倒计时
- 打开“亮屏后不自动恢复联网”后，后续亮屏事件不应单独触发恢复联网

### 7.4 输入控件建议

- 布尔项使用开关
- 秒数使用步进器或数字输入框
- 高风险模式切换使用确认弹窗

## 8. 主题页

### 8.1 页面目标

- 统一控制视觉风格

### 8.2 页面内容

- 跟随系统
- 浅色模式
- 深色模式
- 默认品牌主题
- 备用主题色卡

第一版建议：

- 固定提供 5 套主题
- 不提供自由取色
- 默认品牌主题始终置顶

### 8.3 操作逻辑

切换后：

1. 立即预览
2. 写入偏好存储
3. 全局生效

## 9. 诊断页

### 9.1 页面目标

- 解释“为什么不能工作”
- 集中显示权限、服务、执行器状态
- 支持故障排查

### 9.2 页面结构

#### A. 高级权限状态

显示：

- Shizuku 状态
- ADB 接入状态
- Root 状态
- 当前选中的执行器
- 当前执行器回退状态

命名建议后续统一为：

- `AdbBootstrapped`

#### B. 系统权限状态

显示：

- 使用情况访问
- 通知权限
- 忽略电池优化
- 自启动建议
- 后台无限制建议

#### C. 运行时状态

显示：

- 前台服务是否存活
- 当前规则引擎状态
- 当前待执行任务
- 最近错误

#### D. 调试入口

- 重新检测环境
- 导出日志
- 清空运行日志

### 9.3 操作逻辑

点击“重新检测环境”时：

1. 重新检查所有权限与执行器可用性
2. 刷新所有状态卡片
3. 若检测到核心执行器失效，提示返回接入页

点击“清空运行日志”时：

1. 弹确认框
2. 仅清空日志，不清空规则

点击“导出日志”时：

1. 默认导出结构化 `json`
2. 默认包含最近 500 条日志
3. 包含包名、执行器、规则命中、结果和错误信息
4. 不包含无必要的原始系统调试输出

- 屏幕状态
- Wi-Fi 状态
- 蓝牙状态
- 命中的规则
- 执行动作
- 执行器
- 结果

### 10.3 操作逻辑

- 支持按时间倒序查看
- 支持筛选：
  - 仅成功
  - 仅失败
  - 仅飞行模式
  - 仅移动数据

## 11. 关键全局流程

### 11.1 首次启动流程

1. 启动 App
2. 检查高级接入方式
3. 检查使用情况访问权限
4. 若任一硬门槛不满足，进入接入页
5. 若可用，初始化本地配置
6. 执行首次应用扫描
7. 进入首页

### 11.2 应用安装 / 卸载流程

1. 系统广播触发
2. 后台刷新应用数据库
3. 更新自动识别结果
4. 同步联网列表中的自动项
5. 保留用户已设定的白名单 / 黑名单记录
6. 若当前页面是应用页，触发界面刷新

### 11.3 启动目标应用流程

1. 识别当前前台应用变化
2. 先判断是否为黑名单应用
3. 若在黑名单中，则不触发恢复联网
4. 若不在黑名单，再判断是否在联网列表中
5. 若命中，检查 Wi-Fi 环境例外规则
6. 若未被例外规则拦截，则执行恢复联网
7. 写入日志

### 11.4 离开目标应用流程

1. 识别前台应用变化
2. 判断是否从目标应用离开
3. 若是，检查“离开目标应用后延迟断网”
4. 检查 Wi-Fi 环境例外规则
5. 若允许断网，则开始倒计时
6. 倒计时结束再次确认当前状态
7. 执行断网
8. 写入日志

### 11.5 息屏断网流程

1. 收到息屏事件
2. 检查息屏规则是否启用
3. 检查当前是否处于 Wi-Fi 环境且配置了例外
4. 若允许执行，开始息屏倒计时
5. 倒计时结束后再次确认屏幕仍为熄灭状态
6. 执行断网
7. 写入日志

### 11.6 亮屏 / 解锁流程

1. 收到亮屏或解锁事件
2. 更新运行时屏幕状态
3. 取消不再适用的息屏倒计时
4. 检查是否配置为“亮屏后不自动恢复联网”或“解锁后不自动恢复联网”
5. 第一版默认不因亮屏或解锁本身直接恢复联网
6. 后续是否恢复联网，仍由目标应用命中规则决定

### 11.7 状态保留流程

在执行飞行模式或移动数据切换前：

1. 读取当前 Wi-Fi 状态
2. 读取当前蓝牙状态
3. 执行联网控制动作
4. 按用户设置决定是否恢复 Wi-Fi 状态
5. 按用户设置决定是否恢复蓝牙状态
6. 写入最终结果

## 12. 冲突与异常处理

### 12.1 白名单与黑名单冲突

建议规则：

- 同一应用不能同时处于白名单和黑名单
- 新操作覆盖旧状态
- 黑名单优先阻止自动扫描重新加入联网列表

### 12.2 执行器失效

例如：

- Shizuku 已停止
- Root 授权被撤销
- ADB 环境失效

处理逻辑：

1. 停止自动执行
2. 首页显示异常状态
3. 诊断页显示原因
4. 引导用户重新接入

### 12.3 状态恢复失败

例如：

- 飞行模式已切换成功
- Wi-Fi 或蓝牙恢复失败

处理逻辑：

1. 日志标记为部分成功
2. 首页最近动作显示警告
3. 诊断页展示失败原因

## 13. 后续实现建议

开发顺序建议：

1. 先实现首启接入页和首页
2. 再实现应用页和白黑名单操作
3. 再实现规则页及设置存储
4. 再实现诊断页和日志系统
5. 最后接运行时服务与执行器
